Usage Guidelines

Elevation Hierarchy

Use elevation to create visual hierarchy. Elements with higher elevation appear closer to the user and should be more important or interactive.

Interactive Feedback

Increase elevation on hover and decrease on active/pressed states to provide clear interactive feedback to users.

Accessibility

Ensure sufficient contrast between elevated elements and backgrounds. Elevation should supplement, not replace, other visual cues.

Performance

Use elevation sparingly on mobile devices and consider reducing shadow complexity for better performance.

Implementation

CSS Custom Properties

/* Apply elevation using CSS custom properties */
.my-card {
  border-radius: var(--elevation-border-radius);
  border: var(--elevation-border);
  background: var(--elevation-background);
  box-shadow: var(--elevation-medium);
  transition: var(--elevation-transition);
}

.my-card:hover {
  box-shadow: var(--elevation-medium-hover);
}

Utility Classes

/* Use predefined elevation classes */
<div class="elevation-small">Button</div>
<div class="elevation-medium">Card</div>
<div class="elevation-large">Modal</div>

/* With interactive states */
<button class="elevation-small hover:elevation-medium">
  Interactive Button
</button>
